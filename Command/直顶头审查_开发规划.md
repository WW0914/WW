## 一、解决方案与需求确认

### 1.1 插件目标

插件用于批量审查一个或多个直顶头实体是否满足以下已确认的设计规则：

1. 直顶头的四个主侧面相对全局/工作坐标系 Z 轴的夹角不得小于 `3.0°`；小于 `3.0°` 的侧面为错误面，插件将该面显示为红色。
2. 直顶头沿 Z 方向的高度不得小于 `30.0 mm`。
3. 直顶头底部平面与 Z 轴应为 `90°`（底面本身垂直 Z 轴）。不满足时，该底面显示为红色。

审查完成后，插件在信息窗口和 Block UI 结果区输出每项规则的实测值、判定结果和原因；发生几何错误时在 NX 图形区将对应错误面标红。

### 1.3 审查坐标与角度定义

- **顶出方向**：当前统一使用工作坐标系（WCS）正 Z 方向 `Z=(0,0,1)`。
- **侧面角度**：对于侧面法向量 `n=(nx, ny, nz)`，定义侧面与 Z 轴夹角：

```text
side_angle = asin(abs(nz) / |n|)
```

  - 当侧面与 Z 轴平行时，`side_angle = 0°`。
  - 当侧面向 Z 方向倾斜/带拔模时，`side_angle > 0°`。
  - 本插件判定：`side_angle < 3.0°` 为不合格。

- **底面角度**：对底面法向量 `nb`，定义：

```text
bottom_normal_angle = acos(abs(nb.z) / |nb|)
bottom_plane_to_z = 90° - bottom_normal_angle
```

  - 界面显示 `bottom_plane_to_z`，用于符合设计人员“底面与 Z 轴为 90°”的表达。
  - 几何实现以 `bottom_normal_angle` 判定；底面垂直 Z 轴时其法向应平行 Z 轴，即 `bottom_normal_angle = 0°`。

> 待确认：是否始终使用绝对角度（忽略 `+Z/-Z` 区别）。当前按绝对值处理，底面朝上或朝下均允许。

## 二、审查规则与算法（待门禁 A 确认）

### R1：四个主侧面的最小倾角

| 项目 | 定义 |
|---|---|
| 规则编号 | `R1_SIDE_ANGLE_MIN` |
| 审查对象 | 直顶头实体的四个主侧面 |
| 标准值 | `3.0°` |
| 判定条件 | 任一主侧面的 `side_angle < 3.0°` 为失败 |
| 失败动作 | 将该侧面设置为红色；记录面标签、实测角度和差值 |
| 通过条件 | 成功识别 4 个主侧面，且每个面角度均 `>= 3.0°` |
| 当前容差 | `0.0°`；按用户当前要求严格判定，后续若有制造公差再补充 |

#### R1 主侧面识别算法

```text
输入：用户选择的 Body、WCS Z 轴。
输出：4 个主侧面，或“无法识别”的结构化错误。

1. 遍历 Body.Faces，只保留平面面（Planar Face）。
2. 计算每个候选面的面积、面中心点和单位法向量 n。
3. 计算 side_angle = asin(abs(n.z))，范围为 0° 至 90°。
4. 排除底面/顶面候选：side_angle 大于“侧面候选上限”（建议 30°）的平面不作为主侧面。
5. 按面积由大到小排序，提取形成外轮廓且方向分布合理的 4 个最大候选平面。
6. 逐面执行 side_angle < min_side_angle 判定。
7. 若候选面少于 4 个、超过 4 个且无法唯一筛选，返回“无法可靠识别四个主侧面”，不进行猜测性着色。
```

#### R1 输出示例

```text
[R1] 侧面 1：实测 2.501°，标准 >= 3.000°，失败，已标红。
[R1] 侧面 2：实测 3.002°，标准 >= 3.000°，通过。
[R1] 主侧面数量：4，识别成功。
```

> 待确认：主侧面是否一定为 4 个平面；若直顶头带圆角、台阶、避空或局部成型面，需要补充“哪个面属于主侧面”的实例截图或 PRT。

### R2：Z 向最小高度

| 项目 | 定义 |
|---|---|
| 规则编号 | `R2_HEIGHT_MIN` |
| 审查对象 | 用户选择的直顶头实体 |
| 标准值 | `30.0 mm` |
| 测量方法 | `height = z_max - z_min`，其中 `z_max/z_min` 为实体沿 WCS Z 轴的精确极值 |
| 判定条件 | `height < 30.0 mm` 为失败 |
| 失败动作 | 在结果列表记录失败；实体高亮方式待确认 |
| 通过条件 | `height >= 30.0 mm` |

> 待确认：高度错误没有唯一“错误面”。建议第一版仅将实体整体加红/高亮并输出数值；若只允许“错误面标红”，则需要指定应标红底面、顶面还是两者。

### R3：底面与 Z 轴垂直

| 项目 | 定义 |
|---|---|
| 规则编号 | `R3_BOTTOM_PERPENDICULAR_Z` |
| 审查对象 | 直顶头最低 Z 端的主底面 |
| 标准值 | 底面与 Z 轴 `90.0°` |
| 几何等价条件 | 底面法向量与 Z 轴夹角为 `0.0°` |
| 当前容差 | `±0.1°`（待确认） |
| 判定条件 | `abs(bottom_normal_angle) > 0.1°` 为失败 |
| 失败动作 | 将底面设置为红色；记录实际角度 |
| 通过条件 | 可唯一识别底面，且底面法向满足上述容差 |

#### R3 底面识别算法

```text
1. 遍历实体平面面，计算每个面的 Z 向最低点/面中心点和面积。
2. 从接近实体 z_min 的候选面中，优先选择面积最大的平面作为底面。
3. 读取底面单位法向 nb，计算 bottom_normal_angle = acos(abs(nb.z))。
4. 若 bottom_normal_angle > bottom_angle_tolerance，则将该面标红。
5. 若最低处存在多个面积相近的平面或非平面底部，输出“底面无法唯一识别”，不自动判定。
```

## 二-A、新增规则定义（v0.2，待门禁 A 确认）

### R4：四侧面拔模方向一致性

- 从底面向顶面在每个主侧面上采样；比较该面的外轮廓投影距离是否沿同一方向单调变化。
- 若任一面与其余三面呈相反扩张/收缩趋势，或在顶出方向形成反向斜面，则该面失败并标红。
- 目的：避免“角度数值合格、但拔模方向反向”造成模板配合或顶出卡死。

### R5：四侧面倾角一致性

```text
angle_difference = max(side_angle[1..4]) - min(side_angle[1..4])
angle_difference > max_side_angle_diff -> R5 失败
```

- R5 失败时标红实测角度为最大值和最小值的主侧面，并在结果中显示四个角度与最大差值。

### R6：顶面与底面平行度

```text
top_bottom_parallel_error = angle_between(abs(top_normal), abs(bottom_normal))
top_bottom_parallel_error > top_bottom_parallel_tol -> R6 失败
```

- 顶面为接近实体 `z_max` 的最大平面面；底面为接近 `z_min` 的最大平面面。
- 不平行时将顶面和底面标红。

### R7：Z 向无倒扣 / 无反向斜面

- 将实体沿 Z 方向投影并按 Z 分层采样，检查每个高度层的 XY 截面是否出现与顶出方向相反的局部扩张、回勾或断裂趋势。
- 第一版采取保守策略：能够明确定位到反向平面/反向拔模面时，标红该面；无法可靠识别时只输出“需人工复核”，不作错误着色。
- 说明：该规则依赖真实样件几何校验，待 NX Connector 可用后用正确/错误样件验证采样方向和误报率。

### R9：最小圆角半径（可关闭）

- 用户通过 DLX 半径尺寸控件 `radius_dim0` 输入最小圆角半径；默认值为 `0 mm`，值为 `0` 时跳过 R9。
- 遍历实体中的圆柱面、圆环面及由圆角特征生成的候选面，读取其半径。
- `radius < min_corner_radius` 时失败，并将对应圆角面标红。
- 对倒角、非解析自由曲面、成型曲面不猜测圆角半径；输出“未纳入 R9 自动判定”。

## 三、功能函数分解（草案）

| 编号 | 函数 | 输入 | 输出 | 职责 |
|---|---|---|---|---|
| F1 | `validate_review_input` | Body、内置标准/圆角输入 | 校验结果 | 校验实体存在、阈值有效、WCS 可用 |
| F2 | `collect_planar_faces` | Body | 平面面清单 | 读取面类型、面积、中心、法向与标签 |
| F3 | `find_main_side_faces` | 平面面清单、Z 轴 | 4 个主侧面 | 依据面积、角度和方向分布识别四主侧面 |
| F4 | `measure_side_angle` | 面法向、Z 轴 | 角度 | 计算侧面相对 Z 轴的倾角 |
| F5 | `measure_z_height` | Body、Z 轴 | 高度 | 计算实体的 Z 向高度 |
| F6 | `find_bottom_face` | 平面面清单、Body | 底面 | 识别最低 Z 端主底面 |
| F7 | `check_draft_direction_consistency` | 主侧面、Z 轴 | R4 结果 | 判断四侧面拔模方向是否一致 |
| F8 | `check_side_angle_consistency` | 四个侧面角度、阈值 | R5 结果 | 判断最大倾角差是否超限 |
| F9 | `check_top_bottom_parallelism` | 顶面、底面、阈值 | R6 结果 | 判断顶底面是否平行 |
| F10 | `check_z_direction_undercut` | Body、Z 轴 | R7 结果 | 保守识别顶出方向倒扣 |
| F11 | `check_min_corner_radius` | Body、阈值 | R9 结果 | 检查可解析圆角半径 |
| F12 | `review_geometry` | Body、内置标准/圆角输入 | 审查结果集合 | 依次执行 R1-R7、R9 |
| F13 | `apply_failure_highlight` | 失败面集合 | 着色结果 | 将错误面标红，并保留可恢复信息 |
| F14 | `format_review_report` | 审查结果集合 | 文本/列表行 | 生成信息窗口和 Block UI 可显示内容 |
| F15 | `restore_plugin_colors` | 着色记录 | 恢复结果 | 清除本插件上次设置的红色标记 |

### 函数执行顺序

```text
validate_review_input
  -> collect_planar_faces
  -> find_main_side_faces + measure_side_angle
  -> measure_z_height
  -> find_bottom_face
  -> review_geometry
  -> apply_failure_highlight
  -> format_review_report
```

### 异常与边界处理

| 场景 | 处理方式 |
|---|---|
| 未选择实体 | 不执行审查，提示用户至少选择一个直顶头实体 |
| 实体包含少于 4 个侧面候选 | 输出识别失败，不将任意面误标红 |
| 多实体选择 | 支持一次选择多个实体；逐体执行 R1、R2、R3、R9，报告以实体 Tag 分隔；每个高度不合格实体依次定位、视图缩小至实体适配后的 50%，并弹出 R2 警告 |
| 非毫米单位 | 读取 NX 单位并换算到 mm 后再判定 |
| WCS Z 轴不是顶出方向 | 提示用户调整 WCS；不自动推断顶出方向 |
| 重复审查 | 先恢复本插件上次颜色，再执行新一轮审查 |

### R3 底面误识别回归（2026-07-22）

- **反馈现象**：底面倾斜样件执行 R3 时，插件将一块侧面标红。
- **根因**：原实现仅按平面面最低 Z 值选择底面；直顶头侧面同样延伸到最低 Z，因此可能被错误选为底面。
- **修复口径**：底面候选必须同时满足“平面相对 Z 轴夹角 `>= 60°`”（法向接近 Z 轴）；再在候选中选择最低 Z 的平面。侧面不再参与 R3 底面选择。
- **回归标准**：底面倾斜样件中，只有实际倾斜底面标红；四个侧面不得因 R3 被误标红。
